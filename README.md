# Neon Dungeon Inventory Automation Demo

A proof-of-concept operations workflow for a fictional escape room merch store.

**Live case study:**  
https://addisongillis.github.io/neon-dungeon/

## What this is

Neon Dungeon is a fictional escape room with a merch line: shirts, mugs, and a premium board game.

The project demonstrates how a small business could connect a customer-facing storefront concept, spreadsheet-based inventory rules, and an automation workflow.

The goal is not to build a production checkout system. The goal is to show the business logic, workflow design, and automation pattern clearly.

## What the demo shows

The case study walks through:

- A storefront prototype for Neon Dungeon merch
- A Google Sheets inventory tracker
- Product fulfillment rules
- A board game order represented in inventory
- A Make automation workflow for the operational handoff

## Why it matters

Small businesses often run on fragmented workflows: sales in one place, inventory in another, staff communication somewhere else, and follow-up depending on memory.

This prototype shows how those details can be moved into a clearer system that can be checked, improved, and expanded.
